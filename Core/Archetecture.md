# アーキテクチャ

## 基本方針

- 可能な限りC# に寄せる
  - 最新の.NETに追従する
- グラフィックスなどはC++を使うがC# にバインディングを露出させ、ライブラリとして利用する

## プレイヤービルド

- CoreCLRをベースラインとする
  - Windows, macOS, Linux, (Android) で動作する
- NativeAOT
  - NativeAOTと互換性ある形にしておく
  - リフレクションやIL.Emitの利用に注意
- C++へのトランスパイル
  - コンソール対応
  - ヘビーなので後に回したいが、視野には入れておきたい

## オーサリング

- ランタイムでも必要に応じてオーサリング機能を利用可能にしておく
- 実用的にはオーサリング機能は除外できるように、アセンブリを分割しておく
